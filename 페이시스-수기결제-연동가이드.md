# 페이시스 수기결제 API 연동 가이드

## 1. 개요

페이시스 수기결제 API는 두 가지 인증 방식을 제공합니다. 

| 구분 | 설명 | 필요 정보 |
|------|------|----------|
| **비인증** | 카드정보만으로 결제 | 카드번호, 유효기간 |
| **구인증** | 본인인증 정보 추가 필요 | 카드번호, 유효기간, 비밀번호 앞2자리, 생년월일/사업자번호 |

---

# Part 1. 비인증 수기결제

신용카드번호와 유효기간 정보를 입력하여 결제 승인을 요청합니다.

|---------|------|:----:|:-------:|------|

| mid | String | O | 10 | 상점 ID (페이시스 제공 MID) |

## 4. hashKey 생성 방법

hashKey는 `mid`와 `goodAmt`를 연결한 문자열을 SHA-256으로 해시하여 생성합니다.

### PHP 예시

```php
$mid = "상점ID";
$goodAmt = "10000";
$hashKey = hash('sha256', $mid . $goodAmt);
```

### JavaScript 예시

```javascript
const crypto = require('crypto');
const mid = "상점ID";
const goodAmt = "10000";
const hashKey = crypto.createHash('sha256').update(mid + goodAmt).digest('hex');
```

## 5. 주문번호 생성 규칙

주문번호는 UUID 스타일의 4-4-4-4 형식을 권장합니다.

### 형식
```
[OID]-[YYMM]-[HHMM]-[SSRR]
```

| 세그먼트 | 설명 | 예시 |
|---------|------|------|
| OID | 가맹점 OID (4자리) | `A7K3` |
| YYMM | 년월 (2자리+2자리) | `2512` (2025년 12월) |
| HHMM | 시분 (2자리+2자리) | `1430` (14시 30분) |
| SSRR | 초+랜덤 (2자리+2자리) | `52K7` |

### PHP 생성 코드 예시

```php
function generate_order_number($merchant_oid) {
    $yymm = date('ym');           // 2512
    $hhmm = date('Hi');           // 1430
    $ss = date('s');              // 52
    $rand = strtoupper(substr(md5(microtime()), 0, 2)); // K7
    return "{$merchant_oid}-{$yymm}-{$hhmm}-{$ss}{$rand}";
}
// 결과: A7K3-2512-1430-52K7
```

## 6. 전체 요청 예시 (PHP cURL)

```php
<?php
// 설정값
$apiKey = "발급받은API키";
$mid = "상점ID";
$mkey = "암호화키";
$goodAmt = "10000";

// hashKey 생성
$hashKey = hash('sha256', $mid . $goodAmt);

// 요청 데이터
$data = [
    'ordNo' => 'A7K3-2512-1430-52K7',
    'mkey' => $mkey,
    'mid' => $mid,
    'goodAmt' => $goodAmt,
    'cardNo' => '1234567890123',
    'expireYymm' => '2612',
    'quotaMon' => '00',
    'buyerNm' => '홍길동',
    'goodsNm' => '테스트상품',
    'ordHp' => '01012345678',
    'hashKey' => $hashKey
];

// cURL 요청
$ch = curl_init();
curl_setopt($ch, CURLOPT_URL, 'https://apis.paysis.co.kr:9443/dalgate/api/v1/manual/pay');
curl_setopt($ch, CURLOPT_POST, true);
curl_setopt($ch, CURLOPT_POSTFIELDS, json_encode($data));
curl_setopt($ch, CURLOPT_RETURNTRANSFER, true);
curl_setopt($ch, CURLOPT_HTTPHEADER, [
    'Content-Type: application/json',
    'dal-api-key: ' . $apiKey
]);

$response = curl_exec($ch);
curl_close($ch);

$result = json_decode($response, true);

if ($result['resCode'] === '0000') {
    echo "결제 성공: 승인번호 " . $result['appNo'];
} else {
    echo "결제 실패: " . $result['resMsg'];
}
?>
```

---

## 7. 응답코드 처리

| resCode | 의미 | 처리방법 |
|---------|------|---------|
| 0000 | 정상 승인 | 승인 완료 처리 |
| 그 외 | 비정상 처리 | resMsg 확인 후 오류 처리 |

---

## 8. 주의사항

1. **API KEY 보안**: `dal-api-key`는 서버 측에서만 사용하고, 클라이언트에 노출되지 않도록 합니다.
2. **hashKey 검증**: hashKey는 반드시 서버에서 생성하여 위변조를 방지합니다.
3. **카드정보 보안**: 카드번호, 유효기간 등 민감정보는 SSL/TLS로 암호화하여 전송합니다.
4. **할부개월**: 일시불은 "00", 할부는 "02"~"12" 등으로 입력합니다.
5. **금액 형식**: goodAmt는 숫자 문자열로 전송합니다 (예: "10000", 콤마 없음).

| 항목 | 비인증 | 구인증 |

| 카드번호 | O | O |

## 18. 취소 시 주의사항

1. **원거래 주문번호**: `ordNo`는 결제 시 사용한 주문번호와 동일해야 합니다.
2. **부분취소**: `canAmt`를 원거래 금액보다 작게 설정하면 부분취소가 가능합니다 (PG사 정책에 따라 다름).
3. **취소 기한**: 결제일로부터 일정 기간 내에만 취소 가능합니다 (PG사 정책 확인 필요).
4. **hashKey 불필요**: 취소 API는 hashKey가 필요하지 않습니다.
