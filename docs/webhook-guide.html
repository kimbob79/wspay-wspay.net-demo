결제통보 연동 가이드

결제 승인/취소 시 가맹점 서버로 실시간 알림을 수신하는 방법

1. 개요

결제통보는 결제 승인 또는 취소가 발생했을 때 가맹점 서버로 실시간 HTTP POST 요청을 전송하는 기능입니다.

처리 흐름

결제 발생 → 원성페이먼츠 → 결제통보 발송 → 가맹점 서버
참고: 결제통보는 이벤트 발생 시 즉시 알림을 받는 방식으로, 주기적 API 호출(폴링) 방식보다 효율적입니다.

2. 설정 방법

결제통보를 사용하려면 담당자에게 아래 정보를 전달해 주세요.

항목 설명 예시
수신 URL 알림을 수신할 URL (HTTPS 권장) https://yoursite.com/payment-notify
수신 이벤트 수신할 이벤트 타입 선택 승인, 취소
보안: 수신 URL은 외부에 노출되지 않도록 주의하고, HTTPS를 사용해 주세요.

3. 이벤트 타입

이벤트 event 값 설명
승인 payment.approved 결제가 정상적으로 승인되었을 때
취소 payment.cancelled 결제가 취소되었을 때

4. 데이터 형식

결제통보는 POST 방식으로 JSON 데이터를 전송합니다.

HTTP Headers

Header 값
Content-Type application/json; charset=utf-8
X-Webhook-Event approval 또는 cancel
User-Agent WsPay-Webhook/1.0

JSON 페이로드 (승인 예시)

{
  "event": "payment.approved",
  "version": "1.0",
  "timestamp": "2026-02-05 14:30:52",
  "merchant": {
    "mb_id": "merchant001",
    "mb_name": "테스트가맹점"
  },
  "transaction": {
    "trx_id": "TXN20260205143052",
    "tid": "CATID001",
    "order_number": "ORD-2026-0001",
    "amount": 50000,
    "approval_number": "12345678",
    "approval_datetime": "2026-02-05 14:30:00",
    "cancel_datetime": "",
    "installment": "00",
    "device_type": "terminal"
  },
  "card": {
    "card_name": "신한",
    "card_number": "1234****5678"
  }
}

필드 설명

필드 타입 설명
eventstring이벤트 타입
versionstringAPI 버전
timestampstring발송 시간 (YYYY-MM-DD HH:mm:ss)
merchant.mb_idstring가맹점 ID
merchant.mb_namestring가맹점명
transaction.trx_idstring거래 고유번호
transaction.tidstring단말기 ID (TID)
transaction.order_numberstring주문번호
transaction.amountnumber결제 금액 (원)
transaction.approval_numberstring승인번호
transaction.approval_datetimestring승인 일시
transaction.cancel_datetimestring취소 일시 (취소 시에만, 없으면 빈값)
transaction.installmentstring할부개월 (00: 일시불)
transaction.device_typestring결제 방식 (terminal/keyin)
card.card_namestring카드사명
card.card_numberstring카드번호 (마스킹)

5. 응답 처리

결제통보 수신 후 HTTP 2xx 응답을 반환해야 합니다.

응답 코드 결과 비고
200~299 성공 재시도 없음
400~599 실패 재시도 진행
Timeout 실패 3초 내 응답 필요
권장: HTTP 200과 함께 {"status": "ok"} 또는 빈 응답을 반환하세요.
주의: 응답 지연 시 결제 처리에 영향을 줄 수 있으므로, 복잡한 처리는 비동기로 수행하세요.

6. 재시도 정책

결제통보 발송이 실패한 경우 자동으로 재시도합니다.

항목 값
최대 재시도 횟수 3회 (설정에 따라 변경 가능)
재시도 간격 약 1분
타임아웃 즉시 발송 3초 / 재시도 10초

재시도 흐름

즉시 발송
(3초)		 → 실패 시
pending 저장		 → 크론 재시도
(1분 간격)		 → 최대 3회
초과 시 종료

7. 예제 코드

PHP

<?php
// 결제통보 수신 처리

// 1. JSON 데이터 수신
$payload = file_get_contents('php://input');
$data = json_decode($payload, true);

// 2. 데이터 검증
if (!$data || !isset($data['transaction'])) {
    http_response_code(400);
    exit('Invalid payload');
}

// 3. 이벤트 처리
$transaction = $data['transaction'];
$order_number = $transaction['order_number'];
$amount = $transaction['amount'];

if ($data['event'] === 'payment.approved') {
    // 승인 처리
    update_order_status($order_number, 'paid');
} elseif ($data['event'] === 'payment.cancelled') {
    // 취소 처리
    update_order_status($order_number, 'cancelled');
}

// 4. 성공 응답
http_response_code(200);
echo json_encode(['status' => 'ok']);
?>

Node.js (Express)

const express = require('express');
const app = express();

app.use(express.json());

app.post('/payment-notify', (req, res) => {
    const data = req.body;

    // 데이터 검증
    if (!data || !data.transaction) {
        return res.status(400).json({ error: 'Invalid payload' });
    }

    const { order_number, amount } = data.transaction;

    // 이벤트 처리
    if (data.event === 'payment.approved') {
        console.log(`승인: ${order_number}, ${amount}원`);
    } else if (data.event === 'payment.cancelled') {
        console.log(`취소: ${order_number}`);
    }

    // 성공 응답
    res.status(200).json({ status: 'ok' });
});

app.listen(3000);

Python (Flask)

from flask import Flask, request, jsonify

app = Flask(__name__)

@app.route('/payment-notify', methods=['POST'])
def payment_notify():
    data = request.get_json()

    # 데이터 검증
    if not data or 'transaction' not in data:
        return jsonify({'error': 'Invalid payload'}), 400

    transaction = data['transaction']
    order_number = transaction['order_number']
    amount = transaction['amount']

    # 이벤트 처리
    if data['event'] == 'payment.approved':
        print(f'승인: {order_number}, {amount}원')
    elif data['event'] == 'payment.cancelled':
        print(f'취소: {order_number}')

    # 성공 응답
    return jsonify({'status': 'ok'}), 200

Java (Spring Boot)

import org.springframework.web.bind.annotation.*;
import org.springframework.http.ResponseEntity;
import java.util.Map;

@RestController
public class PaymentNotifyController {

    @PostMapping("/payment-notify")
    public ResponseEntity<Map<String, String>> handleNotify(
            @RequestBody Map<String, Object> data,
            @RequestHeader("X-Webhook-Event") String eventType) {

        // 데이터 검증
        if (data == null || !data.containsKey("transaction")) {
            return ResponseEntity.badRequest().body(Map.of("error", "Invalid payload"));
        }

        @SuppressWarnings("unchecked")
        Map<String, Object> transaction = (Map<String, Object>) data.get("transaction");
        String orderNumber = (String) transaction.get("order_number");
        Number amount = (Number) transaction.get("amount");

        // 이벤트 처리
        String event = (String) data.get("event");
        if ("payment.approved".equals(event)) {
            System.out.println("승인: " + orderNumber + ", " + amount + "원");
            // updateOrderStatus(orderNumber, "paid");
        } else if ("payment.cancelled".equals(event)) {
            System.out.println("취소: " + orderNumber);
            // updateOrderStatus(orderNumber, "cancelled");
        }

        // 성공 응답
        return ResponseEntity.ok(Map.of("status", "ok"));
    }
}

9. FAQ

Q. 결제통보가 오지 않아요.

아래 사항을 확인해 주세요.

  • 수신 URL이 외부에서 접근 가능한지 확인
  • 방화벽에서 원성페이먼츠 서버 IP 허용 여부
  • HTTPS 인증서가 유효한지 확인
  • 관리자 페이지에서 결제통보 설정이 '활성' 상태인지 확인

Q. 중복 알림이 올 수 있나요?

네, 네트워크 문제로 인해 드물게 중복 발송될 수 있습니다. transaction.trx_id 또는 transaction.approval_number를 키로 사용하여 중복 처리를 방지해 주세요.

Q. 발송 이력을 확인할 수 있나요?

판매자센터 > 결제통보 이력 메뉴에서 발송 이력과 상태를 확인할 수 있습니다.

Q. 테스트 모드가 있나요?

담당자에게 요청하시면 테스트 발송을 진행해 드립니다.
