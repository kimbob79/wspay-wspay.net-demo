수기결제 API 연동 가이드

가맹점 서버에서 직접 카드 키인(Keyin) 결제를 요청하는 API

1. 개요

수기결제 API는 가맹점 서버에서 카드 정보를 직접 전송하여 비대면 결제를 처리하는 서버 간(Server-to-Server) API입니다.

지원 기능

기능설명Method
결제카드 키인 결제 요청POST
내역 조회결제 내역 목록 조회GET

처리 흐름

가맹점 서버 → 선샤인페이 API → PG사 결제 처리 → 결과 반환
결제통보 연동: 결제 성공 시 가맹점 서버로 실시간 웹훅 알림을 받으려면 결제통보 연동 가이드를 참고하세요.

2. 인증

모든 API 요청에는 아래 2개의 HTTP 헤더가 필요합니다.

Header설명예시
X-API-Key 발급받은 API 키 (64자) ssp-a3f8c91b2d4e...
X-TID 터미널 ID (PG사별 상이) wspay001

API 키 발급

관리자 페이지 > 가맹점관리 > Keyin 설정 화면에서 "API 키 발급" 버튼을 통해 발급받습니다.

보안 주의: API 키는 서버 측에서만 사용하세요. 클라이언트(브라우저, 앱)에 노출하면 안 됩니다.
API 키가 노출된 경우 즉시 재발급하세요. 기존 키는 즉시 무효화됩니다.

3. 요청 제한

항목제한
분당 요청 수10건 (결제 + 조회 합산)
1회 결제 한도Keyin 설정에 따라 상이
일일 결제 한도Keyin 설정에 따라 상이
월 결제 한도Keyin 설정에 따라 상이
결제 가능 시간Keyin 설정에 따라 상이
주말/공휴일Keyin 설정에 따라 허용/차단

분당 요청 수 초과 시 HTTP 429 응답이 반환됩니다.

4. 결제 요청

POST /api/v1/keyin/pay.php

Request Headers

Header값
Content-Typeapplication/json
X-API-Key발급받은 API 키
X-TID터미널 ID

Request Body (JSON)

파라미터타입필수설명
amountnumber필수결제금액 (100원 이상)
goods_namestring필수상품명
buyer_namestring필수구매자명
card_nostring필수카드번호 (15~16자리 숫자)
expire_yymmstring필수유효기간 (YYMM, 예: 2612)
installmentstring필수할부개월 ("00"=일시불, "02"~"12")
buyer_phonestring선택구매자 전화번호 (루트업/섹타나인/윈글로벌 필수)
buyer_emailstring선택구매자 이메일 (윈글로벌 필수)
cert_pwstring선택카드 비밀번호 앞 2자리 (구인증 시 필수)
cert_nostring선택주민번호 앞6자리 또는 사업자번호 10자리 (구인증 시 필수)

요청 예시

curl -X POST https://wspay.net/api/v1/keyin/pay.php \
  -H "Content-Type: application/json" \
  -H "X-API-Key: ssp-a3f8c91b2d4e6f7a8b9c0d1e2f3a4b5c6d7e8f9a0b1c2d3e4f5a6b7c" \
  -H "X-TID: wspay001" \
  -d '{
    "amount": 50000,
    "goods_name": "테스트상품",
    "buyer_name": "홍길동",
    "buyer_phone": "01012345678",
    "card_no": "1234567890123456",
    "expire_yymm": "2612",
    "installment": "00"
  }'

응답 필드 설명

필드타입설명

data.card_issuerstring카드 발급사
data.pgstringPG사 코드
data.receipt_urlstring영수증 URL

실패 응답 - PG 거절 (HTTP 200)

{
  "success": false,
  "error_code": "PG_DECLINED",
  "message": "결제에 실패했습니다: 카드 한도 초과",
  "pg_error_code": "9999"
}

실패 응답 - 검증 오류 (HTTP 400)

{
  "success": false,
  "error_code": "LIMIT_EXCEEDED",
  "message": "1회 결제한도를 초과했습니다. (한도: 500,000원)"
}
참고: PG 거절은 HTTP 200으로 반환됩니다. success 필드로 성공/실패를 판별하세요.
인증 오류(401), 요청 제한(429) 등만 4xx HTTP 코드를 사용합니다.

8. 연동 예제

PHP

<?php
$api_key = 'ssp-your-api-key-here';
$tid     = 'your-tid-here';
$api_url = 'https://wspay.net/api/v1/keyin/pay.php';

$data = [
    'amount'       => 50000,
    'goods_name'   => '테스트상품',
    'buyer_name'   => '홍길동',
    'buyer_phone'  => '01012345678',
    'card_no'      => '1234567890123456',
    'expire_yymm'  => '2612',
    'installment'  => '00'
];

$ch = curl_init($api_url);
curl_setopt_array($ch, [
    CURLOPT_POST           => true,
    CURLOPT_POSTFIELDS     => json_encode($data),
    CURLOPT_RETURNTRANSFER => true,
    CURLOPT_TIMEOUT        => 30,
    CURLOPT_HTTPHEADER     => [
        'Content-Type: application/json',
        'X-API-Key: ' . $api_key,
        'X-TID: ' . $tid
    ]
]);

$response = curl_exec($ch);
$http_code = curl_getinfo($ch, CURLINFO_HTTP_CODE);
curl_close($ch);

$result = json_decode($response, true);

if ($result['success']) {
    echo '결제 성공! 승인번호: ' . $result['data']['approval_number'];
    echo ' 주문번호: ' . $result['data']['order_no'];
} else {
    echo '결제 실패: ' . $result['message'];
    echo ' 에러코드: ' . ($result['error_code'] ?? '');
}
?>

10. FAQ

Q. 비인증과 구인증의 차이는?

비인증: 카드번호 + 유효기간만으로 결제. cert_pw, cert_no 불필요.
구인증: 카드번호 + 유효기간 + 비밀번호 앞2자리 + 주민번호 앞6자리(또는 사업자번호 10자리) 필요.

Q. PG사별로 필수 파라미터가 다른가요?

PG사buyer_phonebuyer_email
페이시스선택불필요
루트업필수불필요
섹타나인필수불필요
윈글로벌필수필수

Q. API 키가 노출되었습니다.

즉시 관리자에게 연락하여 재발급을 요청하세요. 기존 키는 즉시 무효화됩니다.

Q. 결제 결과 알림(웹훅)을 받고 싶습니다.

관리자에게 결제통보 설정을 요청하세요. API 결제와 일반 결제 모두 동일한 웹훅으로 알림이 전송됩니다.

Q. 취소는 어떻게 하나요?

현재 API를 통한 취소는 지원하지 않습니다. 판매자센터에서 직접 취소하거나 관리자에게 요청하세요.

Q. 테스트 카드번호가 있나요?

PG사별 테스트 카드번호는 담당자에게 별도 문의해 주세요.
